C 언어에서 헤더 파일은 프로그램이나 라이브러리가 다른 코드와 상호 작용할 때 사용되는 선언과 매크로 정의를 포함하는 파일입니다. 헤더 파일은 일반적으로 `.h` 확장자를 가지며, 다음과 같은 목적으로 사용됩니다:

1. **함수 선언**: 헤더 파일은 함수의 원형(prototype)을 포함하여, 다른 파일에서 해당 함수를 사용할 수 있게 해줍니다. 이는 컴파일러에게 함수의 존재와 타입을 미리 알려줌으로써, 함수가 정의되기 전에 사용될 수 있도록 합니다.

2. **매크로 정의**: 매크로는 상수나 함수와 같은 코드 조각을 대체하기 위해 사용되며, 헤더 파일에서 정의될 수 있습니다. 이를 통해 코드의 재사용성과 유지 보수성을 높일 수 있습니다.

3. **타입 정의 및 구조체**: 사용자 정의 데이터 타입이나 구조체(struct)를 정의하여 여러 파일에서 공유할 수 있습니다.

4. **전역 변수 선언**: 헤더 파일은 전역 변수의 선언을 포함할 수 있으나, 일반적으로 이는 권장되지 않습니다.

헤더 파일을 사용하면 코드의 모듈성과 재사용성이 향상되며, 큰 프로젝트를 더 쉽게 관리할 수 있습니다. C 프로그램에서는 `#include` 지시문을 사용하여 헤더 파일을 포함시킵니다. 예를 들어, 표준 입력과 출력을 다루는 `<stdio.h>` 헤더 파일은 `printf` 함수나 `scanf` 함수와 같은 기본적인 입출력 함수를 제공합니다.
